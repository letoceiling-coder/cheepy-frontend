# Проверка на сервере: редирект /person → /auth

## Что проверено по SSH

### 1. Ответ сервера на /person
```bash
curl -sI https://siteaacess.store/person
```
**Результат:** `HTTP/1.1 200 OK`  
Сервер отдаёт страницу с кодом 200, **редиректа на уровне HTTP нет**.

### 2. Nginx
- Конфиг: `/etc/nginx/sites-enabled/siteaacess.store`
- **root:** `root /var/www/siteaacess.store/dist;`
- **location /:** `try_files $uri $uri/ /index.html;`  
Редиректов в nginx для `/person` нет.

### 3. Бандл на сервере
- Файл: `dist/assets/index-JxIBwLO6.js`
- В бандле найдены вхождения **`/auth`** только в:
  - `onClick:()=>h(p?"/account":"/auth")` — кнопка в шапке (клик)
  - `to:t?"/account":"/auth"` — пункт «Профиль» в мобильном меню (ссылка)
  - `href:"/auth"`, `to:"/auth"` — ссылки (Link/кнопки)
  - `onClick:()=>{...,a("/auth")}` — кнопки в модалке и сайдбаре (клик)
- Условного редиректа при загрузке страницы (типа «если pathname === /person → navigate("/auth")») в бандле **нет**.
- В коде есть только редирект в **админку**: в бандле есть `admin/login` (AdminAuthGuard).

---

## Вывод

- На сервере **нет** редиректа для `/person`: ответ 200, nginx не редиректит.
- В **текущем бандле** на сервере (`index-JxIBwLO6.js`) **нет** автоматического перехода на `/auth` при открытии `/person`; `/auth` только в обработчиках клика и в ссылках.

Значит редирект, который видишь в браузере, даёт **старый закэшированный JavaScript** у клиента (или CDN/прокси перед клиентом), а не сервер и не текущий бандл на сервере.

---

## Что сделать

1. **Жёсткое обновление:** на странице https://siteaacess.store/person нажать **Ctrl+Shift+R** (или Cmd+Shift+R).
2. **Проверка в инкогнито:** открыть тот же URL в режиме инкогнито (без расширений).
3. **DevTools → Network:** включить «Disable cache», перезагрузить страницу и убедиться, что подгружается именно **index-JxIBwLO6.js** (как в `dist/index.html` на сервере). Если подгружается другой `index-*.js` — браузер использует старый кэш.
4. **Service Worker:** F12 → Application → Service Workers → для siteaacess.store нажать Unregister, затем обновить страницу.

После этого редирект с `/person` на `/auth` должен пропасть, так как сервер и актуальный бандл редирект не делают.
