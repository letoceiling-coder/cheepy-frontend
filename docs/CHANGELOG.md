## [2.1.0] - 2026-03-16

### Добавлено
- Деплой фронта: **`npm run deploy:frontend`** — локальная сборка + загрузка `dist/` на сервер в `/var/www/siteaacess.store/dist` (скрипт `scripts/deploy-frontend-upload.cjs`).
- Правило в `.cursorrules`: всегда выполнять деплой после изменений; рекомендуемый способ для фронта — build + deploy:frontend.
- Документация: обновлены `docs/DEPLOYMENT.md`, `docs/06_DEPLOYMENT.md`, `docs/ROUTES.md`, `docs/03_FRONTEND_ARCHITECTURE.md`, `docs/README.md`, `docs/audit/AUTH_REDIRECT_AUDIT.md`.

### Изменено
- Навигация: кнопка «Профиль» / «Кабинет» в Header и MobileBottomNav всегда ведёт на `/account` (без редиректа неавторизованных на `/auth`).
- Защита маршрутов: только `/admin/*` требует авторизации (редирект на `/admin/login`); `/account/*`, `/person/*`, `/cart` и остальные — публичные.
- Документация по маршрутам и деплою приведена в соответствие с текущим поведением.

## [2.0.0] - 2026-02-25

### Добавлено
- ✨ Реальные логотипы брендов через `simple-icons` (Nike, Zara, Adidas, H&M, Uniqlo)
- ✨ Плавные переходы между страницами с `framer-motion`
- ✨ Автоматическая прокрутка вверх при навигации (`ScrollToTop`)
- ✨ Страница списка всех брендов `/brand`
- ✨ Страница списка всех продавцов `/seller`
- ✨ Drag-to-scroll для горизонтальных слайдеров
- ✨ Новые секции на главной странице:
  - Лучшие продавцы (по рейтингу)
  - Проверенные продавцы
  - Новые продавцы
  - Популярные продавцы
  - Как сделать заказ
  - Гарантии и доверие
  - Подписка на рассылку
  - FAQ (часто задаваемые вопросы)
  - CTA блоки (регистрация, обращения)

### Изменено
- 🔄 Оптимизирован компонент `BrandLogo` для tree-shaking
- 🔄 Улучшена навигация breadcrumbs на страницах брендов и продавцов
- 🔄 Обновлена структура данных брендов в `marketplaceData.ts`

### Технические улучшения
- 📦 Добавлен пакет `framer-motion@11.x` для анимаций
- 📦 Добавлен пакет `simple-icons@16.9` для логотипов
- 🎨 Создан кастомный хук `useDragScroll` для drag-scrolling
- 🏗 Рефакторинг структуры роутинга с `AnimatePresence`

## Формат

Changelog следует принципам [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

Типы изменений:
- `Добавлено` - новые функции
- `Изменено` - изменения в существующем функционале
- `Устарело` - функции, которые будут удалены
- `Удалено` - удаленные функции
- `Исправлено` - исправления багов
- `Безопасность` - исправления уязвимостей
