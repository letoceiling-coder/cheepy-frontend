# Как настроен сервер и какую сборку отдаёт

## Что проверено на сервере

### 1. Какой root у nginx
- **Активный конфиг:** `/etc/nginx/sites-enabled/siteaacess.store`
- **root:** `root /var/www/siteaacess.store/dist;`  
→ nginx отдаёт **только** каталог **dist/** (сборку), не корень репозитория.

### 2. Что лежит в dist (эту сборку и видит браузер)
- **index.html:** `dist/index.html` (1782 байт) — подключает **`/assets/index-JxIBwLO6.js`**
- **Бандл:** `dist/assets/index-JxIBwLO6.js` — один главный JS-файл приложения

Сервер по запросу `/person` отдаёт **200** и этот же **dist/index.html** → браузер грузит **index-JxIBwLO6.js**. Другой сборки nginx для siteaacess.store не отдаёт.

### 3. Лишний index.html в корне репо (исправлено)
В **корне** репозитория на сервере был второй файл:
- **Путь:** `/var/www/siteaacess.store/index.html` (1688 байт)
- **Содержимое:** `<script src="/src/main.tsx"></script>` — это **dev-**точка входа (исходники), не сборка.

Он **не** отдавался nginx (root у нас `.../dist`), но мог путать при ручных проверках или если бы root когда-то поменяли.

**Что сделано:** файл переименован в **`index.html.source.dev`**, чтобы его нельзя было случайно использовать как главную страницу.

### 4. Кэш браузера (на скриншоте)
- В Network видно: **index-JxIBwLO6.js** — **(memory cache)**, **Disable cache** выключен.
- То есть браузер мог брать **старую версию** этого же файла из кэша (например, с прошлой сборки, где ещё был редирект).

Итог: сервер отдаёт **одну** сборку — из **dist/** с бандлом **index-JxIBwLO6.js**. Редирект при открытии `/person` в текущем коде этого бандла нет; если редирект всё ещё есть у тебя, значит выполняется **закэшированная** старая версия JS.

---

## Что сделать в браузере

1. Включи **Disable cache** в DevTools (вкладка Network).
2. Сделай **жёсткое обновление:** Ctrl+Shift+R (или Cmd+Shift+R).
3. Открой **https://siteaacess.store/person** и проверь, что в Network запрашивается именно **index-JxIBwLO6.js** (и по возможности — что ответ пришёл с сервера, а не из cache).
4. Либо открой ту же ссылку в **режиме инкогнито** (без расширений).

Чтобы сервер явно запрещал кэшировать HTML (и браузер каждый раз подтягивал актуальный `script src`), в nginx для siteaacess.store можно добавить:

```nginx
location = /index.html {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
}
```

(в блок `server` с `server_name siteaacess.store`). В репозитории это уже есть в **deploy/nginx-siteaacess.conf**; на сервере нужно добавить вручную в `/etc/nginx/sites-enabled/siteaacess.store` и выполнить `nginx -t && systemctl reload nginx`.)

## Кратко

| Вопрос | Ответ |
|--------|--------|
| Какую сборку отдаёт сервер? | Из каталога **`/var/www/siteaacess.store/dist`** (nginx root). |
| Какой бандл в этой сборке? | **`index-JxIBwLO6.js`** (указан в `dist/index.html`). |
| Есть ли другая сборка/точка входа? | Нет. В корне репо был лишний **index.html** с `/src/main.tsx` — он переименован в **index.html.source.dev** и не используется. |
| Почему мог быть редирект после «очистил кэш»? | Браузер мог отдавать **закэшированную** старую версию **index-JxIBwLO6.js**. Нужно Disable cache + жёсткое обновление или инкогнито. |
